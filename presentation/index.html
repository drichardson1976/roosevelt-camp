A Little Background

I run a summer basketball day camp

Every summer, parents sign up their kids, counselors volunteer their time, and I coordinate the whole thing.

Registration forms. Emergency contacts. Payment tracking. Pod assignments. Schedule management. It's a lot to juggle.

The Idea

"What if we just...
built it ourselves?"

I'm not a professional software developer. My daughter Audrey is 16. But we had an idea and a new kind of tool:

Claude — an AI that writes code.

We describe what we want in plain English. The AI writes the code. We test it, give feedback, and iterate. Like having a developer on-call 24/7.

The Real Talk

What working with AI
is actually like

It's amazing at...

  • + Turning your ideas into working features in minutes
  • + Handling the tedious, repetitive parts
  • + Remembering every detail (if you teach it how)
  • + Working at 11 PM when inspiration strikes

You still need to...

  • ! Know what you want to build
  • ! Test everything like a real user
  • ! Give clear, specific feedback
  • ! Make product decisions (AI won't do that for you)

The Secret Sauce

We wrote an instruction manual
for the AI

Think of it like onboarding a new employee. We created a document that tells the AI:

  • - How we like things organized
  • - What mistakes to avoid (we learned the hard way)
  • - How to track versions and changes
  • - Always push updates so Audrey and I stay in sync
  • - An 18-item "Never Forget" checklist

The AI reads this before every work session. It's like giving your assistant a perfect memory.

The Process

Keeping It Real

Things broke.
A lot.

x

The "one giant file" disaster

We put everything in one file until it was 12,000 lines long. Splitting it broke five things at once.

x

The "typing makes the cursor disappear" bug

Every form was unusable for a day. Took hours to figure out why — the page was refreshing itself on every keystroke.

x

The "invisible database crash"

Restructured the database wrong. Lost access to parent accounts for 10 versions until we figured it out.

Every failure taught us something. We documented each fix so the AI would never make the same mistake twice.
