The Bug That Taught Us React

Input Focus Loss

The Problem

Every time you typed a character in a form, the cursor would jump away. You'd have to click the input again after every keystroke.

The Root Cause

Form state lived in a giant parent component. Each keystroke re-rendered the entire page, destroying and recreating the input.

The Fix

Extract every form into its own component with local state. Call back to parent only on submit, not on every keystroke.

The Hardest Bug

When Your Variable Name
Overwrites a Library

Babel transforms const supabase = into var supabase =. At the top scope, var supabase is the same as window.supabase.

So creating a Supabase client overwrites the Supabase library itself. The second client creation fails because the library no longer exists.

// The fix: save the library first

const _supabaseLib = window.supabase;

const supabase = _supabaseLib.createClient(...);

Took 3 versions to diagnose. The "Copy Results" button on our test page let us share failures with Claude to finally crack it.
